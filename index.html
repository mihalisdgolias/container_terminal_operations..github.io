What is a Container Terminal?

A container terminal is a facility where cargo containers are transshipped between different transport modes — ships, trains, and trucks. It is a critical node in the global supply chain, requiring precise coordination of vessels, equipment, people, and information systems.

Operations are controlled by a Terminal Operating System (TOS)ⓘ (e.g. Navis N4, SPARCS, TBA OPUS Terminal), which plans vessel stowage, crane sequences, yard allocation, and gate movements in real-time across a rolling 24–72 hour horizon.

The terminal divides into three functional zones: Seaside (berths, STS cranes), Yardside (storage, RTG/RMG/SC cranes), and Landside (gate, road/rail hinterland). Every KPI, every scheduling problem, and every dimensioning formula in this resource maps to one of these three zones.

Problem Definition

Assign each incoming vessel a berthing position (where along the quay) and a service start time (when handling begins), subject to physical, operational, and contractual constraints, so that a chosen objective — typically minimising total waiting time or weighted turnaround time — is optimised.

The BAP is equivalent to a 2D space-time bin-packing problem: the horizontal axis is quay position (metres), the vertical axis is time, and each vessel is a rectangle whose width = LOA and height = service duration. No two rectangles may overlap.

Two fundamental dimensions of classification
Berth layout model
Continuous: quay treated as a line of length L; any position valid. More realistic, harder to solve.
Discrete: quay divided into fixed berth slots. Simpler; misses position flexibility.
Arrival knowledge model
Static: all vessels and their ETAs known before the plan is made. Ideal for periodic replanning.
Dynamic: vessels arrive stochastically; plan updated in a rolling horizon as new ETAs come in.
Objective Functions
  • Min WTMinimise total (or weighted) vessel waiting time — time from anchorage arrival to berth start. Most common objective.
  • Min TATMinimise total turnaround time (waiting + service). Favoured by shipping lines; directly linked to fleet utilisation.
  • Min makespanMinimise time until all vessels in the planning window have departed. Favoured for batch planning.
  • Max util.Maximise berth utilisation. Can conflict with min WT — high utilisation leads to queuing.
  • WeightedPriority-weighted objectives: larger vessels, premium-service customers, or time-sensitive cargo assigned higher weights.
Key Constraints
  • DraftVessel draught ≤ berth depth. Restricts which vessels can use which berth sections.
  • LOAVessel length ≤ available quay segment. Mega-vessels (>350m LOA) can only use certain sections.
  • CranesNumber of cranes reachable from assigned position ≥ minimum required for contracted service time.
  • Tidal windowSome berths (especially in tidal ports) are only accessible within specific tide windows. Adds hard time-of-day constraints.
  • PriorityContracted vessels have guaranteed berthing windows. Spot calls must fit around contracted slots.
  • HazmatVessels carrying dangerous goods may require minimum separation from adjacent berths (IMDG segregation rules).
  • Quay strengthMaximum distributed load (t/m) limits on which berths can serve fully loaded mega-vessels.
Solution Methods
Exact Methods
  • MIPMixed Integer Programming. Optimal but only tractable for small instances (≤15 vessels). Used as a benchmark.
  • B&BBranch-and-Bound. Enumerates the solution tree. Impractical for real terminals without tight bounds.
  • B&PBranch-and-Price. Column generation improves tractability; used in academic BACAP models.
Heuristics & Rules
  • FCFSFirst-Come, First-Served. Simple and fair; default in most real TOS systems. Sub-optimal but robust.
  • SPTShortest Processing Time first. Minimises average wait; risks starving large vessels.
  • PriorityContracted vessels pre-assigned; remaining capacity allocated to spot calls via FCFS or SPT.
  • Rolling horizonPlan is rebuilt every T hours using latest ETAs. Standard approach for dynamic BAP in practice.
Metaheuristics
  • GAGenetic Algorithm. Population-based; explores large solution spaces. Widely used in BAP literature (Kim & Moon 2003).
  • SASimulated Annealing. Single-solution; accepts worsening moves probabilistically to escape local optima.
  • TabuTabu Search. Maintains a "forbidden" list of recent moves to avoid cycling.
  • PSOParticle Swarm Optimisation. Swarm-based; effective for continuous BAP formulations.
Space-Time Diagram — Continuous BAP

Each vessel is a rectangle in (quay position, time) space. Rectangles cannot overlap. The planner's task is to pack all rectangles with minimum vertical displacement (= waiting time) from each vessel's ETA line.

V4 arrives at hour 3 but berth 0–83m is occupied by V1 until hour 8 — incurring 5h waiting time. The planner's goal is to minimise the total area of all waiting brackets.

Real-World Practice
  • Advance Berth RequestVessels submit cargo manifest 24–72h before arrival. Terminal assigns a berth window; shipping line updates stowage plan accordingly.
  • Contracted windowsMajor liner customers (e.g. Maersk, MSC) receive guaranteed berthing windows in terminal service agreements (TSAs). Spot calls fill remaining capacity.
  • TOS berth plannerNavis N4, SPARCS, TBA OPUS Terminal — all include a berth planning module. Most use rolling horizon FCFS + manual planner intervention, not formal optimisation.
  • Replan triggersVessel delay >2h, equipment breakdown, cargo cutoff changes, weather. Each trigger initiates a partial replan of the affected berth window.
  • Pre-arrival alignmentBerth plan drives yard pre-positioning: containers for the vessel must be staged near the assigned berth 2–4h before arrival.

Problem Definition

Given a vessel at berth with a set of bays (numbered positions along the ship from bow to stern) and a number of assigned cranes, determine the sequence and assignment of bays to cranes so that total completion time (or crane idle time) is minimised.

The defining physical constraint is non-crossing: since cranes run on the same rail along the quay wall, they cannot pass each other. At any point in time, cranes must maintain their left-to-right order. This is what makes the problem hard — a crane cannot "skip" to a more urgent bay if another crane is in the way.

Key decision variables
  • AssignmentWhich crane handles which bay (or subset of bays)
  • SequenceOrder in which a crane visits its assigned bays
  • TimingStart time for each bay-crane pair (accounts for crane travel time between bays)
Inter-crane interference (ICI)

When a faster crane catches up with a slower neighbour, it must wait — wasting productive time. ICI is the primary reason adding a 6th crane at 58% utilisation yields only marginal gains vs 5 cranes at 65%. Each additional crane increases ICI for all others.

Objective Functions
  • Min makespanMinimise time from first lift to last lift on the vessel. Most common — directly determines vessel departure time.
  • Min total timeMinimise sum of completion times across all bays. Distributes work evenly.
  • Min idle timeMinimise total crane waiting time (due to ICI and hatch sequencing constraints).
Key Constraints
  • Non-crossingCranes cannot pass each other on the rail. Crane i is always left of crane i+1.
  • Safety marginMinimum distance between adjacent cranes (typically 1 bay). Prevents simultaneous work in adjacent bays.
  • Bay precedenceSome bays must be completed before others (hatches must be opened top-down; deck containers before below-deck).
  • Hatch restrictionsEach hatch cover must be removed before containers underneath can be accessed. Adds precedence constraints between bays.
  • Crane capacityMaximum load per crane per move. Typically 65–80t SWL. Overweight containers require special lifting gear.
Solution Methods
  • MIP / CPMixed Integer Programming and Constraint Programming. Exact; tractable for small vessels (<20 bays).
  • Dynamic programmingOptimal for special cases (e.g. 2 cranes, no ICI). Polynomial time.
  • Beam searchTruncated tree search; good quality solutions quickly. Used in commercial TOS systems.
  • GA / SAStandard metaheuristics. GA representations typically encode crane-to-bay assignment + sequence.

Problem Definition

Determine the placement of every container on a vessel across multiple port calls, so that (a) the vessel remains stable and structurally sound throughout the voyage, (b) containers are accessible in the correct sequence at each port, and (c) crane operations at each port are efficient.

The SSP is inherently a multi-port, multi-stage problem: a decision made at Port A determines what reshuffles are required at Port B. Good stowage plans look ahead across the entire rotation.

Two-stage decomposition (standard approach)
Stage 1 — Master stowage plan (by shipping line)
Allocates container groups (bay/tier ranges) to each port of call. Done 24–72h before arrival. Optimises for multi-port stability and efficiency.
Stage 2 — Slot planning (by terminal planner)
Assigns individual containers to specific slots within the allocated groups. Must respect weight limits, hazmat segregation, reefer plug points, and OOG clearances.
Stability Constraints
  • GM > minMetacentric height must exceed the minimum positive value throughout the voyage. If GM is too low the vessel rolls dangerously; too high causes uncomfortable snap-rolling.
  • TrimLongitudinal balance — difference in draught between bow and stern. Typically −0.5m to +1.5m. Affects propeller efficiency.
  • ListLateral heel angle. Must be within ±0.5° under SOLAS. Caused by asymmetric loading across port/starboard.
  • Bending momentLongitudinal stress on the hull girder. Hogging (sag amidships) and sagging must stay within class limits.
  • Shear forceLocal structural loads between adjacent holds. Heavy containers adjacent to empty holds create high shear.
Operational Constraints
  • Discharge sequenceContainers for the next port must be on deck or at the top of holds — not buried under boxes destined for later ports (avoids overstow).
  • IMDG segregationDangerous goods classes must maintain minimum separation distances (e.g. Class 1 explosives cannot be adjacent to Class 3 flammable liquids).
  • Reefer plug pointsRefrigerated containers must be placed within cable reach of electrical sockets. Number of sockets per bay limits reefer concentration.
  • Weight limitsCell guide weight capacity per stack, maximum stack weight on hatch covers, and IMO weight distribution rules all constrain container placement.
  • OOG clearanceOut-of-gauge cargo needs overhead and lateral clearance; cannot be placed under cranes with insufficient outreach.
Impact on Terminal Operations
  • Overstow movesEach overstow requires lifting and temporarily placing a container before accessing the target box. Directly reduces crane utilisation. Poor stowage = lower effective GCR.
  • Crane assignmentStowage plan determines which bays need service and in what sequence — directly feeds into the CAP. A well-structured plan enables parallel crane deployment with minimal ICI.
  • Yard pre-positioningTerminal yard manager needs the confirmed stowage plan to pre-stage containers near the assigned berth. Late plan changes force yard reshuffles.
  • Port rotation efficiencyContainers correctly positioned for the next port reduce crane moves per call at downstream ports — a network-level efficiency gain.
Data Exchange — BAPLIE

The BAPLIE (Bay Plan / Stowage Information) message is the EDIFACT standard for transmitting container stowage plans between shipping lines and terminals. It contains:

  • Container IDISO 6346 number — owner code, serial, check digit
  • Slot positionBay-row-tier designation (e.g. 042284) for every container
  • Weight & typeVGM, ISO type code (20GP, 40HC, 45R1, etc.)
  • Port of loading/dischargeUN/LOCODE for each container's origin and destination
  • Hazmat infoIMDG class, UN number, flashpoint — required for dangerous goods
Terminal TOS systems import BAPLIE on vessel arrival to populate the working stowage plan. Any late changes after cutoff must be transmitted as BAPLIE updates — a major source of disruption.
Why Integration Matters

The BAP, CAP, and Stowage Planning are solved sequentially in most real terminals — berth first, then cranes, then stowage. But they are deeply interdependent: the berth assignment determines which cranes are reachable; the crane assignment determines service time (which feeds back into the berth plan); and the stowage plan determines the sequence of crane operations. Solving them jointly yields measurably better solutions.

BAP + CAP (BACAP)

Joint berth and crane assignment. The crane assignment determines service duration, which is needed to set the berthing time window. Most commercial TOS systems solve these jointly on a 24–48h rolling horizon.

BAP + CAP + Stowage (BACASP)

The most comprehensive formulation. Stowage determines crane sequences; crane sequences affect service times; service times feed the berth plan. Correcher et al. (2019) solved this with column generation for the first time.

Yard pre-positioning

Often overlooked in research. The berth+stowage plan must be communicated to the yard manager to pre-stage containers near the correct berth 2–4h before arrival. Late changes cascade through yard crane schedules.

Key Takeaways for Terminal Design
  • 🔗
    Solve BAP and CAP jointly. Sequential solving leaves significant turnaround time on the table — crane assignment changes service duration, which changes the optimal berthing time.
  • 📋
    Stowage quality directly impacts GCR. Each overstow reduces crane utilisation. A structured stowage plan for parallel crane deployment can reduce berth time by 15–25%.
  • ⏱️
    Rolling horizon > one-shot planning. Vessel ETAs change. A plan rebuilt every 6–12 hours using latest AIS data consistently outperforms a static 48h plan.
  • 🚪
    Gate TAT is the hidden driver. IAPH (2016): Truck TAT explains 42% of CMPH variation. Gate automation and TAS improve quayside productivity without touching berth or crane planning.
  • 📦
    Coordinate with the yard planner. The berth + stowage plan must drive yard pre-positioning 2–4h before arrival. Late plan changes cascade into yard crane reschedules and delay vessel departure.

Problem Definition (CSAP)

Assign each arriving container to a specific yard slot (block, bay, row, tier) so that future retrieval requires the fewest crane moves. The Container Storage Assignment Problem is solved continuously as containers arrive at the gate or are discharged from vessels.

The fundamental trade-off is unavoidable: tight grouping by retrieval sequence minimises future reshuffles but makes it harder to fill all slots evenly; random/loose assignment maximises utilisation but causes many reshuffles at retrieval.

Decision variables
  • BlockWhich yard block receives the container (determines equipment type and distance to berth)
  • Bay / RowPosition within the block — affects future crane travel distance
  • TierStack height — higher tiers cause reshuffles when lower containers are needed first
Objectives
  • Min reshufflesExpected number of non-productive relocations at retrieval time — primary objective
  • Min travelTotal crane travel distance at retrieval — assign export containers to blocks adjacent to their loading berth
  • Max utilisationFill slots evenly across all blocks — target 65–75%. Above 85% reshuffles grow exponentially.
  • Balance workloadDistribute crane tasks evenly across blocks to avoid hotspot congestion
Constraints
  • Block capacityTGS × stacking height per block cannot be exceeded
  • Equipment zonesReefer containers → reefer-capable slots only. OOG → designated oversized areas.
  • Hazmat separationIMDG class separation distances must be maintained — see Special Cargo tab
  • Vessel sequenceExport containers for the same vessel should be co-located to minimise crane travel at loading
  • Dwell estimateContainers with long expected dwell should not block short-dwell containers in a stack

Solution Methods
  • MIPExact for small instances. Used as benchmark. Impractical at terminal scale in real time.

Two Related Sub-Problems
YCAP — Yard Crane Assignment Problem

Which crane serves which block during a given time window. Determines crane deployment across the yard — a tactical decision made every 1–4 hours.

YCSP — Yard Crane Scheduling Problem

The sequence of individual (source slot, destination slot) moves within a block assignment. An operational decision, updated continuously as tasks are completed or priorities change.

Non-crossing constraint

Multiple RTGs on the same block rail cannot pass each other — exactly analogous to the non-crossing constraint in the Quay Crane Scheduling Problem. A crane cannot skip to a higher-priority task if another crane is between it and the target slot. This makes the problem hard even with just 2 cranes per block.

170-ft minimum separation (Petering & Murty 2009)

Two RTGs on the same block must maintain at least 170 ft (≈52m, ~8 container slots) of separation at all times. Violating this creates truck traffic jams at the transfer zone that halve effective YCR. This constraint is enforced by the TOS in real-time.

Work Order Generation

The TOS generates a prioritised task list for each crane — a sequence of (pick-up slot, delivery slot) pairs. Priority ordering:

  • 1st — EDDVessel-bound containers sorted by vessel departure deadline (Earliest Due Date first). Starvation of quay cranes is the highest-cost failure mode.
  • 2nd — AppointmentsTruck-pickup containers sorted by TAS appointment time. Late truck = detention charges.
  • 3rd — HousekeepingPre-marshalling, block rebalancing, and space-making moves. Deferred if vessel work is pending.
Key Concepts
  • Dual-cyclingCrane handles a discharge task and a load task in a single traversal of the block — halves dead (unladen) travel. Requires TOS coordination with the berth plan to ensure the right container is available for pick-up on the return pass.
  • Intra-block tasksMoves within a single block — the vast majority of yard crane work. Crane does not leave the block.
  • Inter-block tasksMoves between blocks — requires ITV handoff. Crane drops container at transfer zone; ITV carries it to another block.
  • Gantry travel timeCrane travel time between slots is significant — acceleration and deceleration profiles matter for accurate scheduling. Typical peak speed: 130–150m/min (gantry), 70–100m/min (trolley).
Solution Methods
  • EDD dispatchPractical standard. Fast, vessel-priority aware. Misses crane travel optimisation.
  • Rolling horizon MIPOptimal for 1–4 hour window. Used in automated terminals (ASC systems).
  • Beam searchTruncated tree search. Good quality quickly. Standard in commercial TOS crane modules.
  • GA / SAResearch benchmarks. GA encodes crane-task assignment + sequence.

The Reshuffle Problem

When a target container is buried under others in a stack, those containers above must be temporarily relocated before the target can be picked up. Each relocation is a non-productive move — it consumes crane capacity without advancing throughput.

RESHFP = non-productive relocations ÷ productive moves
  • Target<0.2 — fewer than 1 reshuffle per 5 productive moves
  • At 75% util.RESHFP typically 0.2–0.3
  • At 85% util.RESHFP spikes to 0.5+ as stacks become dense and retrieval order uncertain
Itsuro Accessibility Index (S)

S is the probability that a randomly chosen container in a stack can be directly accessed without moving any container above it.

  • S = 1.0Stack height h=1 — every container always accessible. Zero reshuffles possible.
  • S ≥ 0.6Minimum acceptable for import containers. Retrieval order unpredictable — needs headroom.
  • S ≥ 0.5Minimum for export / transship. More predictable retrieval sequence allows tighter stacking.
  • S decreases as h ↑The key design trade-off with stacking height — higher h increases TEU capacity but degrades accessibility.
Design implication: The stacking height values h used in the Dimensioning formulas are not simply "maximum possible" — they are the highest h consistent with maintaining S above its minimum. This is why Chu & Huang (2005) specifies different h values for import vs export containers.
Offline Reshuffle — Pre-marshalling

Deliberate physical rearrangement of export containers into vessel loading sequence, performed before the vessel arrives — typically overnight or during a quiet period.

  • TimingTypically starts 8–16h before vessel arrival. Completed before the vessel berths.
  • Crane costTypically 2–6 crane-hours per vessel call — time that could otherwise serve import pickups.
  • Benefit40–70% reduction in live reshuffles during loading. Directly improves GCR during vessel service.
  • Depth of sortDecision variable: sort by bay only? By row? Full sort to exact load sequence? Deeper sort = more crane time but fewer live reshuffles.
  • Trade-offPre-marshalling consumes crane capacity. If the vessel call changes size at cutoff, some pre-marshalling work is wasted. Must be balanced against other pending crane tasks.
Online Reshuffle — Reactive

Unplanned relocations that occur during live crane operations when a buried container must be accessed. Disruptive because they interrupt the planned work order queue.

  • TriggerTOS requests a specific container; another container blocks access to it.
  • EffectCrane must place the blocking container somewhere (reshuffle destination); other tasks in the queue must wait.
  • Cascade riskThe reshuffle destination may itself cause a future reshuffle — exponential degradation at high yard utilisation.
  • Minimised byGood initial storage assignment + pre-marshalling. Both are cheaper than live reshuffles.
The Slot Planning Problem

Before assigning a container to a slot, the TOS can estimate the expected number of future reshuffles for every candidate placement. Assigning a container with an early retrieval date deep in a tall stack is expensive; assigning a long-dwell container to the top of a stack near the exit lane is efficient.

Good assignment

Short-dwell container → top tier, near exit lane, adjacent to assigned berth. Zero expected reshuffles.

Acceptable assignment

Long-dwell container → tier 2, block utilisation 60%. Low expected reshuffles because yard has headroom.

Bad assignment

Early-departure vessel export → tier 4 (bottom), block at 85% utilisation. High probability of reshuffles — TOS should reject and find a better slot.

Dispatch Problem

Given N quay cranes generating tasks and M ITVs available, assign ITVs to crane tasks to minimise crane waiting time. The ITV cycle time — from receiving a task, picking up the container, delivering it, and returning empty — determines how many ITVs are needed per crane.

ITV cycle time components
  • Travel ladenDrive from quay crane transfer zone to yard crane transfer zone (with container)
  • Yard waitWait for yard crane to pick up / put down container at the stack
  • Travel unladenReturn to quay crane transfer zone (empty)
  • Quay waitWait for quay crane to load / unload (buffer zone)
Dual-cycling

An ITV that has delivered a discharge container to the yard picks up an export container for the same quay crane before returning — halving dead (empty) travel. Requires TOS to coordinate crane schedule with ITV routing. Can increase ITV effective throughput by 20–35%.

Buffer Zone Sizing

A buffer zone at the waterside transfer point allows the quay crane to continue working while ITVs are in transit. Without a buffer, the quay crane stops every time an ITV is late — directly reducing GCR.

  • Rule of thumb2–4 buffer positions per quay crane. Larger buffer decouples crane from ITV variability but takes up apron space.
  • Optimal sizeFunction of ITV cycle time variance and number of ITVs per crane. Solved analytically using queueing theory (M/G/c model).
AGV-Specific Challenges
  • DeadlockTwo AGVs blocking each other on a one-way road segment. Prevented by conflict-free routing algorithms (reservation-based or zone-based).
  • Road network designOne-way loops with passing bays. Network topology directly affects deadlock probability and average travel time.
  • Battery rechargingBattery AGVs must periodically leave service for charging (30–60 min). Fleet must be oversized to maintain coverage. Opportunity charging at transfer zones emerging.
  • Mixed traffic exclusionHuman workers cannot enter AGV operating zones. Creates operational zoning constraints that affect maintenance access.

📭 Empty Container Management
  • Volume15–25% of yard capacity. Empties are shipped for repositioning — the global empty container imbalance costs the industry $20B+/yr.
  • Stacking height5–8 high (lightweight — no structural weight stack limit). Very space-efficient but requires careful ISO type sorting.
  • SortingMust be sorted by ISO type (20GP, 40HC, 45R1 etc.) and shipping line for efficient retrieval and inspection.
  • Storage locationTypically placed at the rear of the yard (lowest real estate value) or in an off-dock empty depot to free prime yard space.
  • Repositioning revenueShipping lines pay terminals for empty repositioning moves — a meaningful ancillary revenue stream.
  • InspectionPre-trip inspection (PTI) required before empties are accepted — damage affects the shipping line's equipment pool.
🌡️ Reefer Management
  • Constant powerReefer containers require permanent electrical connection at all times — they cannot be placed in non-plug slots, even temporarily.
  • Plug densityTypically 10–15% of yard slots are reefer-capable. This caps the maximum reefer volume the terminal can handle simultaneously — a hard operational constraint.
  • Temperature monitoringTOS tracks set-point vs actual temperature for every reefer. Alarms automatically dispatched to reefer technician on call. Required by cargo owners and shipping lines.
  • PTIPre-Trip Inspection conducted 24h before vessel loading to verify refrigeration unit integrity, door seals, and temperature log. Failures require repair or cargo transfer.
  • Stacking limitTypically limited to 2–3 high — heavier units and need for individual plug management makes high stacking impractical.
  • Energy costReefers are a major terminal electricity cost. Power management systems regulate voltage and load-shed non-critical plug points during grid peaks.
⚠️ Hazmat Segregation (IMDG)
  • IMDG Classes1 Explosives · 2 Gases · 3 Flammables · 4 Solid flammables · 5 Oxidisers · 6 Toxic · 7 Radioactive · 8 Corrosives · 9 Misc dangerous
  • Class 1 (Explosives)Isolated corner with 50m minimum separation from all other dangerous goods. Requires dedicated fencing, signage, fire suppression, and 24h security.
  • Class 3 + Class 8Flammables and corrosives cannot be stored adjacent — IMDG segregation table specifies "Separated from" for these combinations.
  • ManifestTerminal must maintain a real-time hazmat manifest available to the port authority, fire brigade, and customs at all times.
  • Inspection priorityHazmat containers are priority flagged for customs physical inspection — creates a dedicated inspection lane and pad requirement in the yard.
  • Yard designHazmat zone requires dedicated vehicle access routes (to avoid routing hazmat through populated areas of the yard), spill containment bunds, and separation from reefer electrical infrastructure.

Optimal Block Length — Petering & Murty (2009)

Simulation across 432 scenarios (4 terminal sizes × 2 equipment levels × 54 block lengths) shows a concave GCR-vs-block-length relationship. The optimal range is consistent across all tested configurations.

  • <28 slotsOnly 1 YC can effectively operate per block — too short for multi-crane deployment.
  • 56–72 slotsOptimal. Balances YT routing flexibility and YC–YC interference. Somewhat longer than average real-world block.

YC Deployment rule

Restrictive deployment (min/max YCs per block) consistently outperforms free deployment in all 34 tested configurations. The advantage grows with terminal size and when equipment is scarce. Coordination beats adding equipment.

Row Length

What is a CFS?

A Container Freight Station (CFS) serves as an intermediate location between the marine terminal and the final destination of cargo. It plays a critical role in the logistics chain by facilitating the efficient handling, storage, and processing of containers and their contents — particularly for LCL (Less than Container Load)ⓘ shipments.

📦
Consolidation
Combining multiple smaller shipments (LCL) into a single container for export. Reduces per-unit shipping costs for small shippers who cannot fill a full container.
📂
Deconsolidation
Unpacking imported containers and distributing contents to respective consignees. Essential for importers receiving goods from multiple suppliers in one container.
🏪
Storage
Temporary storage for containers and their contents. Securely holds cargo until transport to final destination. Reduces pressure on terminal yard dwell times.
🛃
Customs Clearance
Provides space for customs inspections, documentation, and regulatory procedures. Acts as a bonded warehouse for goods pending duty assessment.
🔧
Cargo Handling
Loading, unloading, sorting, and repacking of cargo. Includes repalletisation, relabelling, and reconfiguration to meet specific delivery requirements.
✅
Quality Control
Inspections and quality checks ensuring cargo meets required standards. Particularly important for food products, pharmaceuticals, and regulated goods.
CFS Benefits
⚡
Efficiency

Streamlines handling and processing of cargo, reducing delays and improving overall logistics efficiency throughout the supply chain.

🔄
Flexibility

Provides shippers and consignees flexible options for cargo consolidation, interim storage, and last-mile distribution.

💰
Cost Savings

Reduces transportation costs by consolidating smaller shipments into full container loads (FCL), optimizing freight economics.

Dimensioning

How to calculate terminal capacity, storage area, and yard requirements — Chu & Huang (2005), UNCTAD, and supply/demand approaches

Demand-Oriented Dimensioning

Given a throughput target, work backward to determine infrastructure requirements — yard area, berths, and cranes.

Annual throughput (T)
TEUs per year — the primary demand driver. Separate by cargo type: import, export, transshipment.
Dwell time by container status
Import: 4–7 days | Export: 2–4 days | Transship: 1–3 days. UNCTAD PPS 2024 overall median: 3.1 days. Dwell time is the single strongest driver of yard demand.
Stacking height (h)
Higher stacking reduces footprint but degrades accessibility. Practical limits: SC 1-over-3 (h=3), RTG 1-over-5 (h=4–4.5), RMG 1-over-6 (h=4.5). Use Itsuro accessibility index S ≥ 0.6 for import.
Peak factor (P)
P = peak volume ÷ average volume. Typically 1.2–1.5. Terminal must be sized for peak demand.
→ Required TGS and yard area
Output: required ground slots and total terminal footprint. Cross-validate with berth capacity.
Demand-Oriented TGS Formula
TGSreq = (T × P × D̄) ÷ (365 × h)
  • TAnnual throughput (TEU/yr)
  • PPeak factor (typically 1.2–1.5)
  • hAverage stacking height (tiers)
  • D̄Weighted average dwell (days) = Hi·Di + He·De + Ht·Dt
  • Hi,e,tShare of imports / exports / transshipment (must sum to 1)
  • Di,e,tDwell time for imports / exports / transshipment (days)
Worked Example — SHP 513 Problem 7
T = 600,000 TEU/yr  |  P = 1.35  |  h = 4
D̄ = 0.45 × 5 + 0.40 × 3 + 0.15 × 4 = 2.25 + 1.20 + 0.60 = 4.05 days
TGS = (600,000 × 1.35 × 4.05) ÷ (365 × 4)
     = 3,280,500 ÷ 1,460
     = 2,247 ground slots required
Key Inputs & Design Rules
Dt, Di, De
Dwell times for transshipment, import, and export containers (days). Halving dwell ≈ doubling throughput capacity.
Ht, Hi, He — stacking heights
Tiers for transshipment, import, export. Higher stacking reduces footprint but raises reshuffle factor. Maintain accessibility index S ≥ 0.6 (import).
Peak Factor (P)ⓘ
Peak volume ÷ average volume. Range 1.2–1.5. Design for peak.
TEU Ground Slots (TGS)ⓘ
Fundamental yard capacity unit. Area = TGS × m²/slot (typically 14–16 m² per slot).
IAR (Interchange Area Req.)
SC 1-berth: 48–72 TEU | SC 2-berth: ~120 TEU | RMG/RTG: 0
⚠️ Practical operation factor
Apply factor 0.75 to Cmax when setting design throughput targets — terminals cannot sustainably operate at theoretical maximum (Chu & Huang 2005).
Supply-Oriented Dimensioning

Given fixed land, compute the maximum throughput that can be handled. Start with the Chu & Huang formulas to get TGS, then apply the capacity formula.

Step 1 — Compute TGS from land

Apply Chu & Huang formulas (see Formulas tab) to derive TGS from yard dimensions and equipment choice.

TEU capacity = TGS × h
  • TGSTotal ground slots from Chu & Huang formulas
  • hAverage stacking height (tiers)
Step 2 — Maximum annual throughput (Cmax)
Cmax = (365 × 2 × TGS) ÷ [m·(Dt/Ht) + (1−m)·(De/He) + (1−m)·(Di/Hi)]
  • mTranshipment ratio (0–1). Hub terminal: m > 0.5
  • Dt,e,iDwell time — transship / export / import (days)
  • Ht,e,iStacking height — transship / export / import (tiers)
  • Factor 2Each TEU counted once in + once out of terminal
Tmax = TEU capacity × 365 ÷ (D̄ × P)
  • D̄Weighted average dwell (days) = simplified denominator above
  • PPeak factor (1.2–1.5)
  • DesignApply 0.75 × Cmax as the practical throughput design target
Inputs & Outputs
  • TGSFrom Chu & Huang — the binding constraint
  • h / Hi,e,tStacking height by cargo type
  • mTranshipment ratio (0–1). Hub: m > 0.5
  • Dt,e,iDwell time by cargo type (days)
  • PPeak factor (1.2–1.5)
  • CmaxTheoretical max capacity (TEU/yr) → multiply by 0.75 for design target
Worked Example — SHP 513 Problem 5
Given: SC terminal, WY=320m, DY=400m, TGS=1,380, h=3. Find: Tmax at D̄=5d and D̄=3d (P=1.3).
TEU capacity = 1,380 × 3 = 4,140 TEU
At D̄ = 5 days: Tmax = 4,140 × 365 ÷ (5 × 1.3) = 232,000 TEU/yr
At D̄ = 3 days: Tmax = 4,140 × 365 ÷ (3 × 1.3) = 387,000 TEU/yr (+67%)
No new infrastructure — dwell reduction alone unlocks the gain.
Key principle: Balance berth and yard

A mismatch — high crane productivity with insufficient yard, or large yard with too few berths — creates systemic bottlenecks. Always check both constraints. The binding one determines terminal throughput.

Chu & Huang Empirical Formulas (2005)

These formulas calculate the number of container rows, crane rows, and total ground slots for a container yard given equipment and layout parameters.

Formula 1 — NC: Containers per Row
NC = (WY − WR − WP − WS) ÷ (LC + CR)
  • NCNumber of containers end-to-end in each row (along quay wall direction)
  • WYWidth of the container yard (= length of quay wall), m
  • WRWidth of perimeter roadways, m
  • WPTotal width of passageways and turning areas, m
  • WSWidth reserved for special needs (reefers, hazmat, etc.), m
  • LCNominal length of a 20-ft container = 6.1 m
  • CRContainer spacing for customs door access = 0.4 m
Formula 2 — NCR: Number of Crane Rows
NCR = DY ÷ (WC + 2·WT + WA)
  • NCRNumber of crane footprints fitting across the yard depth (perpendicular to quay)
  • DYDepth of the container yard (perpendicular to quay), m
  • WCTotal width of handling equipment (crane span including outreach), m
  • WTTravel clearance width per side: SC 1.6 m, RTG 2.0 m, RMG 0 m
  • WAAisle between adjacent crane rows: RMG 1 m, SC/RTG 0 m
Formula 3 — CR: Container Rows per Crane
CR = N × NCR
  • NContainer rows spanned by one crane unit (RTG: 5–6, RMG: 11–12, SC: 1)
  • NCRNumber of crane rows from Formula 2
  • CRTotal container rows aligned parallel to the quay wall
Formula 4 — Total Ground Slots (TGS)
TGS = (NC × CR) − IAR
  • NCContainers per row — from Formula 1
  • CRTotal container rows — from Formula 3
  • IARInterchange area requirement: SC 1-berth 48–72 TEU, SC 2-berth ~120 TEU, RMG/RTG 0 TEU
  • TGSGround-level container positions — the fundamental yard capacity unit. Multiply by h to get total TEU capacity.
Annual Throughput Capacity — Cmax (Itsuro 2001 / Chu & Huang 2005)
Cmax = (365 × 2 × TGS) ÷ [m·(Dt/Ht) + (1−m)·(De/He) + (1−m)·(Di/Hi)]
  • TGSTotal ground slots from Formulas 1–4
  • mTranshipment ratio (0–1)
  • Dt,e,iDwell time by cargo type (days)
  • Ht,e,iStacking height by cargo type (tiers)
  • Factor 2Each TEU counted once in + once out
  • 0.75Practical operation factor — always apply to Cmax for design target
Key insight (Chu & Huang 2005): Annual throughput varies by >100% across crane types at different dwell times, and by up to 60% across crane types at the same dwell. Dwell time is the dominant variable — not crane type or stacking height.

UNCTAD — Container Storage Area Requirements

The UNCTAD Handbook for Port Planners provides standardised formulas for estimating storage area as a function of throughput and dwell time.

A = (T × D × PF) ÷ (365 × H × UTIL × GS)
  • AStorage area required (TEU ground slots)
  • TAnnual throughput (TEU/yr)
  • DAverage dwell time (days)
  • PFPeak factor (typically 1.2–1.5)
  • HStacking height (tiers)
  • UTILYard utilisation target (0.65–0.75)
  • GSGround slots per unit area

The numerator calculates the TEUs in the yard at peak; the denominator converts available ground slots (adjusted for stacking and utilisation) into capacity.

UNCTAD — Berth Capacity Requirements

Berth day requirements calculated on an annual basis, cross-checking whether berth infrastructure matches the cargo demand.

BD = T ÷ (BPR × BE)
  • BDBerth days required per year
  • TAnnual throughput (TEU/yr)
  • BPRBerth productivity rate (TEU/berth-day)
  • BEBerth efficiency factor

Compare BD to available berth days (365 × number of berths × occupancy target) to check if berth is the binding constraint or if yard is.

Practical guidance: Always run both berth and yard calculations in parallel. The smaller capacity is your bottleneck — design around it first.
The Frankel Empirical Formula

An alternative empirical approach for estimating the container storage area required. Incorporates statistical variability in both dwell time and stack height — making it more conservative than deterministic formulas.

CA = [(C × A × (T + 2t))] ÷ [365 × Z × 10⁴ × (H + 2h) + U]
  • CAArea required for storage (sq. m.)
  • CExpected container volume (TEU/yr)
  • AProjected area per container = TGS (sq. m.) — normally 15 sq. m. per TGS
  • TAverage dwell time (days)
  • tStandard Deviation (SD) of dwell time (days)
  • HAverage expected stack height (tiers)
  • hSD of stack height
  • ZStorage utilisation — depends on storage control, rationale, shipment sizes
  • UTotal area utilisation — affected by equipment type, lane widths, gantry span
Key distinction vs UNCTAD formula: Frankel adds 2t to dwell time and 2h to stack height — i.e., adds 2 standard deviations — to ensure the terminal can handle statistical peaks in arrivals and stacking variability, not just the average. This is a more conservative (larger) area estimate.

Selecting the Optimal Handling System

System selection is driven by throughput volume, land constraints, labour costs, and automation ambitions. The IAPH (2016) survey of 34 terminals provides empirical benchmarks.

🔰
Straddle Carrier (SC)

Best for: Single-berth terminals, FCL operations, flexible layouts. Combines transport and stacking in one machine. Best land efficiency at 1-berth scale.

Area: 32.1–35.1 m²/TEU (285–300 TEU/ha)

Typical capacity: 0.5–1.5M TEU/year

🔶
RTG Crane System

Best for: High-volume conventional terminals. Good stacking density. Relocatable between blocks. Dominant system worldwide.

Area: 35.4–43.3 m²/TEU (231–282 TEU/ha) — highest area per TEU

Typical capacity: 1–4M TEU/year

🔷
RMG Crane System

Best for: Automated mega-terminals with constrained land. Highest density. Fixed rails — less flexible but automation-ready.

Area: 32.9–41.2 m²/TEU (250–304 TEU/ha) — better than RTG in large yards

Typical capacity: 2–6M TEU/year

TYM-Based Equipment Sizing — IAPH (2016)

Total Yard Moves (TYM) is a better yard equipment sizing metric than TEU, because it accounts for cargo mix. A terminal with high transshipment generates far fewer yard moves per TEU than a gateway terminal.

TYM = TEUimport+export × 3 + TEUtransship × 1
  • TYMTotal Yard Moves per year — better equipment sizing metric than TEU alone
  • Factor 3Import/export containers: load + unload + empty return = 3 yard moves per TEU

  • RTG sizingRTG count scales with TYM at R²=0.91 vs R²=0.80 with TEU alone (IAPH 2016). Use ~90,000 TYM/RTG as initial estimate.

Optimal Block Length — Petering & Murty (2009)

Simulation experiments across 432 scenarios show that block length has a concave relationship with GCR. Too short: YTs lose routing options. Too long: YC–YC interference increases.

Optimal block length: 56–72 (20-ft) slots

Consistent across all four terminal scenarios tested (small/large × less/more equipment). Somewhat longer than the average block in use today.

  • < 28 slotsOnly 1 YC can effectively operate per block — too short
  • 56–72 slotsOptimal: balances YT routing and YC efficiency
  • > 84 slotsMultiple YCs interfere — diminishing returns
YC Deployment Rule

Use restrictive deployment (min/max YCs per block) rather than free deployment. Restrictive outperforms free in all 34 block-length × fleet-size combinations tested. Advantage is larger at larger terminals and with less equipment. Coordination beats adding equipment.

The 170-ft (8-slot) minimum YC–YC separation rule must be respected to prevent truck traffic jams that halve YC productivity.

Glossary of Abbreviations

All abbreviations and acronyms used in container terminal operations — click any term for a detailed popup explanation

STSⓘ
Ship-to-Shore Crane
Large quay crane that loads/unloads containers directly between vessel and terminal.
RTGⓘ
Rubber Tyred Gantry Crane
Mobile yard crane that straddles container rows on rubber tyres. Can be relocated between blocks.
RMG
Rail Mounted Gantry Crane
Fixed-rail yard crane with high stacking capacity. Enables full automation. Permanent installation.
SC
Straddle Carrier
Single machine combining transport and stacking. Straddles containers; stacks 1-over-2 or 1-over-3.
ITVⓘ
Internal Transfer Vehicle
Tractor-trailer operating inside terminal only, transferring containers between quay cranes and yard.
AGVⓘ
Automated Guided Vehicle
Driverless ITV following magnetic/GPS guidance. Receives work orders directly from TOS.
ASC
Automated Stacking Crane
Automated version of RMG/RTG. Operates without human operators; controlled by TOS/PLC.
FLT
Forklift Truck
Low-capacity container handler. Used for CFS/LCL operations, empty depots, and small volumes.
RS
Reach Stacker
Wheeled crane with telescopic boom. Stacks up to 5–6 high. High flexibility, low capital cost.
SWL
Safe Working Load
Maximum load (in tonnes) a lifting device is rated to handle safely. STS cranes typically 65–80t SWL.
YC
Yard Crane
Generic term for any crane operating in the storage yard (RTG, RMG, or ASC).
PLC
Programmable Logic Controller
Industrial computer controlling crane automation. Receives move orders from TOS and executes them electro-mechanically.
TOSⓘ
Terminal Operating System
Core software controlling all terminal operations: vessel planning, yard management, equipment dispatch, gate control, billing.
PCSⓘ
Port Community System
Neutral digital platform connecting all port stakeholders for information exchange. Data submitted once, reused by all.
MSW
Maritime Single Window
Government system for electronic submission of all vessel arrival/departure declarations. Mandatory under FAL Convention (2024).
TSW
Trade Single Window
National portal for all import/export/transit regulatory requirements. Single entry point for trade documentation.
ANPRⓘ
Automatic Number Plate Recognition
Camera system that reads truck license plates at gate to validate pre-booked appointments automatically.
OCRⓘ
Optical Character Recognition
Camera system at gate that reads ISO 6346 container numbers from all sides as trucks pass through.
TASⓘ
Truck Appointment System
Pre-booking system for truck time slots at the gate. Reduces peak congestion and enables container pre-positioning.
EDIⓘ
Electronic Data Interchange
Computer-to-computer exchange of standard business documents. Maritime standard: UN/EDIFACT.
RFID
Radio Frequency Identification
Tag-based tracking technology. Used on chassis, containers, and trucks for real-time location tracking in the yard.
VTMIS
Vessel Traffic Management Information System
Port authority system tracking vessel movements in the port area, anchorage, and approach channels.
AIS
Automatic Identification System
Mandatory transponder on vessels transmitting position, speed, and identity in real time. Feeds into VTMIS.
ETA
Estimated Time of Arrival
Vessel's predicted arrival time at anchorage or berth. Used to plan crane gangs, yard space, and ITV deployment.
GCRⓘ
Gross Crane Rate
Total moves ÷ total crane hours at vessel (including all idle time). UNCTAD PPS 2024 median: 18.1 lifts/ship-hr.
NCR
Net Crane Rate
Moves ÷ productive crane hours only. Always higher than GCR. Gap between NCR and GCR reveals operational inefficiency.
YCR
Yard Crane Rate
Moves per yard crane per hour. Benchmark: 15–20 for RTG, 25–30 for automated RMG.
DWELLⓘ
Container Dwell Time
Days a container stays in the terminal. UNCTAD PPS 2024 median: 3.1 days. Excess dwell = congestion + demurrage.
TAT
Turnaround Time
Vessel: arrival to departure. Truck: gate-in to gate-out. Both are key efficiency indicators.
GTT
Gate Turn Time
Time from truck arrival at gate to exit. Reflects gate processing efficiency and yard organisation.
BSH
Berth-Ship Hours
Total hours a vessel occupies the berth. Minimising BSH maximises berth throughput capacity.
YUTIL
Yard Utilisation
TEUs stored ÷ total TEU capacity. Planning guideline: 65–75%. Above 85% causes excessive reshuffles.
LSCIⓘ
Liner Shipping Connectivity Index
UNCTAD index measuring a country's integration into the global container shipping network. Top 2025: China, Korea, Singapore.
LPI
Logistics Performance Index
World Bank composite index of logistics efficiency. Countries with PCS/MSW average LPI 3.1 vs 2.7 (UNCTAD 2025).
EBITDA
Earnings Before Interest, Taxes, Depreciation & Amortisation
Operating margin measure. UNCTAD PPS 2024 median: 48% of revenue. Reflects port financial health.
CAPEX
Capital Expenditure
Investment in infrastructure and equipment. Ports are capital-intensive; performance must be assessed over long time horizons.
BAPLIEⓘ
Bay Plan / Stowage Information
EDIFACT message carrying the complete container stowage plan. Exchanged between shipping lines and terminals pre-arrival.
FCLⓘ
Full Container Load
Single shipper fills entire container. Moves as a sealed unit through the terminal — no CFS processing required.
LCLⓘ
Less than Container Load
Multiple shippers share one container. Requires CFS consolidation (export) or deconsolidation (import).
OOGⓘ
Out of Gauge
Cargo exceeding standard ISO container dimensions. Requires flat rack, open top, or platform containers. Stored at block edges.
B/L
Bill of Lading
Contract of carriage between shipper and carrier. Title document for the cargo. Required at gate-out for container release.
CY
Container Yard
The storage area inside the terminal where containers are stacked awaiting vessel loading or customer pickup.
CFS
Container Freight Station
Facility for LCL consolidation/deconsolidation, customs clearance, and value-added services. Located near marine terminal.
TEU
Twenty-foot Equivalent Unit
Standard unit of container capacity. 1 × 20ft container = 1 TEU. 1 × 40ft container = 2 TEU. Used for all throughput and capacity measures.
FEU
Forty-foot Equivalent Unit
40-foot container = 1 FEU = 2 TEU. Most containers today are 40ft or 40ft High Cube.
EDIFACTⓘ
UN/Electronic Data Interchange for Administration, Commerce and Transport
International EDI standard for maritime messages (BAPLIE, COPRAR, COARRI, CODECO, etc.).
IMDGⓘ
International Maritime Dangerous Goods Code
IMO regulation governing sea transport of dangerous goods. 9 hazmat classes with mandatory segregation rules.
VGMⓘ
Verified Gross Mass
SOLAS requirement: mandatory declared weight of packed container before vessel loading. Mandatory since July 2016.
SOLAS
Safety of Life at Sea
IMO international convention setting minimum safety standards for ships. VGM, fire safety, and structural standards all derive from SOLAS.
FAL
Facilitation of International Maritime Traffic
IMO convention (1965, amended 2022). Mandates MSWs and electronic pre-arrival data exchange. Effective January 2024.
IMO
International Maritime Organization
UN agency responsible for safety, security, and environmental performance of international shipping. Publishes SOLAS, IMDG, FAL, etc.
ISO
International Organization for Standardization
Sets container dimension standards (ISO 668, 1496) and the container numbering standard (ISO 6346: owner code + serial + check digit).
WTO TFA
WTO Agreement on Trade Facilitation
Multilateral agreement requiring TSWs and transparent trade information. Countries with PCS/MSW achieve 87% TFA implementation vs 79% without.
PSC
Port State Control
Inspection of foreign vessels in a port's jurisdiction. Can detain ships for safety/IMDG/SOLAS violations. Terminal must not load deficient vessels.
TGSⓘ
TEU Ground Slots
Number of ground-level container positions in a yard. Fundamental capacity unit. Total TEU capacity = TGS × stacking height.
PFⓘ
Peak Factor
Peak period throughput ÷ average throughput. Typical range 1.2–1.5. Terminal must be sized for peaks, not averages.
IAR
Interchange Area Requirement
Yard area reserved for vehicle/crane interchange. SC: 48–72 TEU (1-berth) or ~120 TEU (2-berth). RMG/RTG: 0.
DY
Depth of Container Yard
Yard dimension perpendicular to the quay wall (metres). Determines number of crane rows (NCR). Typical range: 300–600m.
WY
Width of Container Yard
Yard dimension parallel to the quay wall = length of quay wall (metres). Determines NC (containers per row). 300–350m per berth.
WC
Width of Handling Equipment
Total crane span including outreach. SC: 3m | RTG 5+1: 20m | RTG 6+1: 23.5m | RMG 11: 54m | RMG 12: 57m.
NC
Number of Containers per Row
Chu & Huang Formula 1 output. Containers end-to-end along quay wall direction. Depends on WY, WR, WP, LC, CR.
NCR
Number of Crane Rows
Chu & Huang Formula 2 output. How many crane footprints fit across yard depth DY. Depends on DY, WC, WT, WA.
CR
Container Rows per Crane
Chu & Huang Formula 3 = N × NCR. N = container rows the crane spans (RTG: 5–6, RMG: 11–12, SC: 1).
BD
Berth Days
UNCTAD metric: annual throughput ÷ (berth productivity rate × efficiency factor). Used to check if berth is the binding capacity constraint.
QC
Quay Crane
Generic term for any crane working at the quayside loading/unloading vessels. Synonymous with STS. GCR is measured per QC working hour beside a busy berth.
YT
Yard Truck
Internal tractor-trailer (prime mover / hustler / UTR) that transfers containers between the quay cranes and the storage yard. Operates inside the terminal only.
XT
External Truck
Road truck that enters the terminal through the gate to deliver export containers or collect import containers. Subject to gate processing procedures.
ALV
Automated Lift Vehicle
Automated transport vehicle with self-lifting capability. Unlike AGVs, ALVs do not need synchronisation with quay or stack cranes, enabling a decoupled (IS-SOQN) system. Fewer vehicles needed vs AGV for equivalent throughput.
BMPH
Berth Moves Per Hour
Total container moves per hour across all cranes working a vessel. IAPH 2016 survey average: ≈65 BMPH (range 40–140). A terminal-level productivity metric.
CMPH
Crane Moves Per Hour
Moves per individual crane per hour. IAPH 2016 average: ≈27 CMPH (range 10–43). Does not scale with terminal size; the strongest predictor is truck TAT (R²=0.42, IAPH 2016).
TYM
Total Yard Moves
TYM = TEU(import+export) × 3 + TEU(transship) × 1. Better than TEU for equipment sizing: RTG count scales with TYM at R²=0.91 vs R²=0.80 with TEU alone (IAPH 2016).
VSC
Vessel-Size Coefficient
IAPH index: VSC = 1·%(<4k TEU) + 2·%(4–8k TEU) + 3·%(>8k TEU). Score near 3 = ultra-large fleet; near 1 = feeder port. R² vs CMPH ≈ 0.03 — vessel size does not drive crane productivity.
RESHFP
Reshuffle Factor
Non-productive reshuffles ÷ productive moves. Target: <0.2. Rises sharply when yard utilisation exceeds 85% or stacking height is too high relative to the accessibility index S.
WAIT
Vessel Waiting Time
Hours from vessel arrival at anchorage to start of berthing. UNCTAD/Clarksons Dec 2024: 6.4h (developed countries), 10.9h (developing countries). Rising from 2023 levels.
BERTHOCC
Berth Occupancy Rate
Booked berth time ÷ available berth time. No universal benchmark — varies by port strategy. Higher rates raise vessel waiting times; balance depends on landlord vs operator model.
GVOL
Gate Volume
Trucks processed per hour per gate lane. Reflects gate throughput capacity. Varies widely by terminal size, appointment system, and degree of OCR/ANPR automation.
REJRATE
Gate Rejection Rate
Percentage of trucks turned away at gate entry (documentation errors, no appointment, wrong container). Indicator of pre-arrival information quality and TAS effectiveness.
OPEX
Operational Expenditure
Day-to-day operating costs: labour, energy, maintenance, and administration. Contrast with CAPEX. High automation shifts the ratio from OPEX-heavy to CAPEX-heavy operations.
ROI
Return on Investment
Net profit ÷ invested capital. Financial KPI identified by Hinkka et al. (2018) as most important to practitioners but hardest to compute in simulation without real cost data from TOS.
BAP
Berth Allocation Problem
Optimization problem: assign incoming vessels to berths to minimise waiting time and maximise berth utilisation. Largely solved by GA/ILP methods in the literature (largely solved by Period 1, 2010–2015).
CAP
Crane Assignment Problem
Allocation of quay cranes to vessels or bays to minimise turnaround time. Often solved jointly with BAP. Attention shifted post-2015 as the problem was considered largely solved.
HRC
High-Risk Cargo
Cargo classified as hazardous, radioactive, or otherwise posing elevated risk during terminal handling. Covered under Risk & Safety performance category in Kurniawan et al. (2022).
BIM
Building Information Modelling
CAD extension with time (4th dim.) and cost (5th dim.) for infrastructure planning. Currently siloed from terminal simulation and live TOS data — no data interface exists (Hinkka et al., 2018).
CAD
Computer-Aided Design
2D/3D design tool used for terminal layout planning. Forms the basis of BIM. Cannot currently export directly to operational simulation software.
IS-SOQN
Infinite-Server Semi-Open Queuing Network
Analytical queuing model used by Roy et al. (2020) to model ALV fleets. The infinite-server (IS) component captures the decoupled nature of ALVs: the network with V vehicles is equivalent to a GI/G/V queue.
IAPH
International Association of Ports and Harbors
Global port industry association. Published the 2016 Study on Productivity and Key Indicators of Container Terminals covering 34 terminals across 23 ports in 9 regions.
UNCTAD
UN Conference on Trade and Development
UN body publishing the Port Performance Scorecard (PPS) and Review of Maritime Transport (RMT). Source of GCR, dwell time, and vessel waiting time benchmarks used throughout SHP 513.
IHS
IHS Markit (data provider)
Commercial maritime data provider. Abramowski et al. (2018) used IHS Maritime & Trade data on container ships built 2005–2015 to derive regression formulas for ship design parameters.
LBP
Length Between Perpendiculars
Ship design dimension from stem to rudder post (metres). Regression: LBP ≈ 1.50 + 0.413·ln³(TEU). Determines required berth length (berth ≥ 1.1 × LBP). Range: 47.5–383m.
DWT
Deadweight Tonnage
Total weight a vessel can carry including cargo, fuel, and stores (metric tonnes). One of the two primary independent variables in Abramowski et al. (2018) regression equations alongside TEU capacity.
GT
Gross Tonnage
Measure of overall internal volume of a vessel. GT ≈ −1097 + 11.05·TEU (Abramowski et al., 2018). Used for port dues calculation and turnaround time estimation.
PME
Main Engine Total Power
Total installed propulsion power (kW). Range: 404–81,250 kW for container ships. Used to size berth electrical supply infrastructure. Regression: PME ≈ −11,799 + 0.0289·ln⁶(DWT).
Cmax
Maximum Annual Handling Capacity
Chu & Huang (2005) Eq. 6: Cmax = (365 × 2 × TGS) ÷ [m·Dt/Ht + (1−m)·De/He + (1−m)·Di/Hi]. Apply 0.75 practical operation factor to obtain design throughput target.
m
Transhipment Ratio
Proportion of terminal throughput that is transhipment (0–1). Transhipment containers use only 1 yard move vs 3 for import/export. Affects Cmax and TYM. Hub terminals: m > 0.5.
S
Accessibility Index (Itsuro)
S = 1: container always accessible without rehandling. S = 0: never accessible. Practical minimum: S ≥ 0.6 for import, S ≥ 0.5 for export/transship. Drives maximum safe stacking height.
WR
Perimeter Road Width
Width of road surrounding the container yard (metres). SC: WR = 5m. RMG/RTG: WR = 22m. Subtracted from WY in Chu & Huang Step 1 formula.
WP
Apron / Passage Width
Width of operational passage between quay and first yard block (metres). SC 1-berth: 20m | RMG 1-berth: 30m | RTG 1-berth: 44m. Larger for 2-berth configurations.
WT
Truck Lane Width
Width of lane for yard trucks alongside a crane span (metres). SC: 1.6m | RTG: 2.0m | RMG: 0m (rail fixed, no dedicated truck lane width). Used in Chu & Huang NC formula.
WA
Aisle Width
Width of aisle between crane rows (metres). RMG: WA = 1m. SC and RTG: WA = 0m. Used in Chu & Huang Step 1 formula for NC calculation.
LC
Container Length (nominal)
Nominal length of a 20-ft container used in Chu & Huang calculations: LC = 6.1m. Combined with container spacing CR = 0.4m to give NCR = DY ÷ (LC + CR).
CTu
Container Unloading Cycle Time
Roy et al. (2020): CTu = Wq + Tq + Wv + Tv + Ws + Ts. Total time to move one container from vessel to stack (seaside → transport → yard). Binding constraint = resource with highest utilisation Ue.
CTl
Container Loading Cycle Time
Roy et al. (2020): CTl = Ws + Ts + Wv + Tv + Wq + Tq. Same components as CTu in reverse order (stack → transport → vessel). Both are used in integrated queuing network models for terminal design.
